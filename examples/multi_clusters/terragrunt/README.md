# Terragrunt Multi-Cluster Example

This example deploys Zesty across **one AWS account** and **multiple EKS clusters** using Terragrunt with automatic dependency resolution.

## Architecture

```
account/              → applied ONCE per AWS account  (IAM role + policy + zesty_account)
kompass-eks-prod/     → applied per EKS cluster       (Helm release)
kompass-eks-staging/  → applied per EKS cluster       (Helm release)
```

### Dependency Graph (DAG)

```
        account
       (applied first)
        /          \
kompass-eks-prod  kompass-eks-staging
   (parallel)       (parallel)
```

## Directory Structure

```
examples/terragrunt/
├── modules/
│   └── kompass/                          # local Terraform module for Helm release
│       ├── main.tf
│       ├── variables.tf
│       └── versions.tf
└── live/
    ├── root.hcl                          # org, project
    └── prod/
        ├── environment.hcl              # environment = "prod"
        └── aws/
            ├── datacenter.hcl           # remote_state (S3), AWS provider generation
            └── us-east-1/
                ├── region.hcl           # region = "us-east-1"
                └── my-account/
                    ├── account.hcl      # account_id, AWS profile
                    └── zesty/
                        ├── account/
                        │   └── terragrunt.hcl       # calls published module
                        ├── kompass-eks-prod/
                        │   └── terragrunt.hcl       # depends on account
                        └── kompass-eks-staging/
                            └── terragrunt.hcl       # depends on account
```

## Prerequisites

- Terraform >= 1.1
- Terragrunt >= 0.45
- AWS credentials configured (profile in `account.hcl`)
- Zesty API token — set the `token` value in the `generate "zesty_provider"` block inside `account/terragrunt.hcl`
- `kubectl` access to the target EKS clusters

## Usage

### Apply everything (Terragrunt resolves the DAG)

```bash
cd live/prod/aws/us-east-1/my-account/zesty
terragrunt run-all apply
```

Terragrunt will apply `account/` first, then both `kompass-eks-*` directories in parallel.

### Apply individually

```bash
# 1. Account (must be first)
cd live/prod/aws/us-east-1/my-account/zesty/account
terragrunt apply

# 2. Each cluster independently
cd ../kompass-eks-prod
terragrunt apply

cd ../kompass-eks-staging
terragrunt apply
```

### Destroy a single cluster (others untouched)

```bash
cd live/prod/aws/us-east-1/my-account/zesty/kompass-eks-staging
terragrunt destroy
```

## Adding a new cluster

1. Copy any `kompass-eks-*/` directory
2. Rename it (e.g. `kompass-eks-data/`)
3. Change `cluster_name` in `locals` inside `terragrunt.hcl`
4. Run `terragrunt apply`

## Configuration

### `account/terragrunt.hcl`

- Source: published module `zesty-co/aws-eks-cluster/zesty`
- Generates `zesty_provider.tf` with `provider "zesty"` block
- AWS provider generated by `datacenter.hcl`
- Token is set directly in the generated provider block

### `kompass-eks-*/terragrunt.hcl`

- Source: local module at `modules/kompass/`
- Generates `helm_provider.tf` with EKS data sources + Helm provider
- AWS provider generated by `datacenter.hcl`
- Reads `kompass_values_yaml` from `account` dependency
